GIT , GIT/HUB

En este trabajo se muestran los comandos mas destacados a nivel basico de la consola gib,espero te ayude,animo.

GIT INIT
Presentacion al proyecto
GIT ADD .
Enviar a la Zona de ensayo
GIT COMMIT -M"mensaje"
Envia a la zona segura (instantanea)
GIT STATUS -S
Iindica el estado de respaldo de los documentos,
GIT LOG --ONELINE
te muestra todos los commit ,de esa rama
GIT RESET --HARD (codigoversion)
resetear a la version indicada con el codigo
GIT COMMIT -AM"mensaje"
es la version fusionadas del comando add y comit
GIT PUSH origin main
push es para enviar los archivos de local a la nube git hub,en este caso es origin main ,pero podria ser a cualquier otra rama
GIT PULL origin main
pull es para recibir de la nube al local ,en este caso nos descargariamos origin main,pero podria ser cualquier otra rama
GIT TAG nombre version -M"mensaje"
el tag es una etiqueta que se utiliza para poder tener un listado de las versiones terminadas ejem:nombre V1.2
GIT BRANCH nombre rama
se utiliza para crear una rama de pruebas paralela a la main principal,se utiliza para hacer cambios y pruebas ,distribucion del trabajo,etc
GIT CHECKOUT nombre de la rama
se utiliza para moverse entre ramas
GIT MERGE rama que quiero juntar
esta instruccion SE HACE SIEMPRE DESDE LA RAMA MAIN,se utiliza para fusionar la ramas con la main principal
GIT BRANCH -D nombre rama a borrar
esta instruccion SE HACE SIEMPRE DESDE LA RAMA MAIN se utiliza para BORRAR una rama
GIT CLONE url del repositorio
se utiliza para clonar un repositorio de la nube git hub a local
GIT PUSH -U ORIGIN nombre rama
se utiliza para subir las ramas que hemos creado paralelas a la master
GIT BRANCH --LIST
se usa para obtener por consolaa una lista de las ramas
